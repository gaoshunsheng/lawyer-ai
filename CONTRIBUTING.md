# 贡献指南

欢迎来到律师AI助手项目！我们非常欢迎任何形式的贡献，无论是代码、文档、测试用例还是bug报告。

## 🚀 如何开始贡献

### 1. 准备工作
- 确保您有GitHub账户
- Fork本仓库到您的账户
- 克隆您的Fork到本地

```bash
git clone https://github.com/YOUR_USERNAME/lawyer-ai.git
cd lawyer-ai
```

### 2. 设置开发环境

#### 后端开发环境
```bash
# 安装Java 17+和Maven
# 配置后端
cd backend
mvn clean install
```

#### AI服务开发环境
```bash
# 安装Python 3.11+
cd ai-service
pip install -r requirements.txt
```

#### 前端开发环境
```bash
# 安装Node.js 18+
cd frontend
npm install
```

### 3. 创建分支
```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/issue-description
```

## 📝 代码规范

### 提交信息格式
使用约定式提交格式：
```
<类型>[可选 范围]: <描述>

[可选 正文]

[可选 页脚]
```

类型包括：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

示例：
```
feat(chat): 添加多轮对话记忆功能

- 实现对话历史管理
- 添加上下文长度控制
- 优化对话质量

Closes #123
```

### 代码风格

#### Java代码
- 遵循Google Java Style Guide
- 使用4个空格缩进
- 类名使用PascalCase
- 方法名和变量名使用camelCase

#### Python代码
- 遵循PEP 8规范
- 使用4个空格缩进
- 导入顺序：标准库、第三方库、本地模块
- 使用类型提示

#### JavaScript/TypeScript代码
- 使用ES6+语法
- 遵循Airbnb JavaScript Style Guide
- 使用Prettier进行代码格式化

## 🧪 测试

### 编写测试
- 每个新功能都应包含相应的测试
- 测试覆盖率应保持在80%以上
- 测试应独立运行，不依赖外部服务

### 运行测试

```bash
# 运行后端测试
cd backend
mvn test

# 运行AI服务测试
cd ai-service
pytest

# 运行前端测试
cd frontend
npm test
```

## 📚 文档

### 更新文档
- API更改必须更新API文档
- 新功能需要更新用户文档
- 重大变更需要更新架构文档

### 文档格式
- 使用Markdown格式
- 图片放在`docs/images/`目录
- 链接使用相对路径

## 🔧 提交流程

### 1. 代码审查
- 所有更改都需要通过代码审查
- 至少需要一名核心贡献者批准
- 确保代码质量和一致性

### 2. 持续集成
- 确保所有测试通过
- 确保代码格式化正确
- 确保没有引入安全问题

### 3. 合并请求
- 提供清晰的问题描述
- 链接相关issue
- 确保代码是最新的
